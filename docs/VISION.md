# Vision Platform

## Purpose

We are building a KI-gestützte Projektorchestrierungsplattform that turns project context into usable operational and management structure.

The platform should not only store projects and tasks.  
It should help teams understand:
- what needs to be done
- what depends on what
- which side effects and follow-up tasks emerge from context
- where risks, decisions, and escalation points are forming
- which management artifacts should be generated from the current project state

## Vision Statement

The product shall become a structured orchestration layer for complex projects, especially where project plans, communication, documents, governance, and decisions interact.

## Core product promise

The system should:
- provide a shared project core for multiple project types
- support ERP implementations as the first fully deepened project type
- keep construction and software projects structurally prepared from the beginning
- integrate context from documents, emails, and meeting notes
- derive proposals for tasks, risks, dependencies, and decision needs
- provide reviewable and traceable KI-assisted outputs
- create useful operational and management artifacts from the project graph

## What the product is not

The product is not:
- a simple task board
- a document archive without semantic use
- an autonomous black-box planner
- a one-off ERP tool without broader extensibility
- a framework rewrite project

## Product principles

1. Stable core before deep specialization
2. Extensions instead of later rebuilds
3. Context is a first-class input
4. AI writes proposals, not hidden truth
5. Every step must stay explainable
6. Every expansion must honor the fixed target stack — no reintroduction of obsolete implementations
