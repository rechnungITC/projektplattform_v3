# ERP Data Modeling

## Übersicht
Dieser Skill befähigt den ERP-Architekten, komplexe relationale Datenbankstrukturen zu entwerfen, die Geschäftsprozesse effizient abbilden.

## Anwendungsbereiche
- Neuentwicklung von ERP-Modulen
- Migration von Altsystemen
- Anpassung bestehender Datenstrukturen an neue Geschäftsanforderungen

## Kernkompetenzen
- **Entity-Relationship-Modellierung**: Identifikation von Entitäten, Attributen und Relationen.
- **Normalisierung**: Vermeidung von Redundanzen (bis zur 3. Normalform).
- **Indexierungsstrategien**: Optimierung der Abfrage-Performance.
- **Data Governance**: Definition von Constraints und Validierungsregeln.

## Workflow
1. Anforderungsanalyse durchführen.
2. Konzeptionelles Datenmodell erstellen (ER-Diagramm).
3. Logisches Datenmodell ableiten.
4. Physisches Datenmodell für das spezifische DBMS (z.B. PostgreSQL, Oracle) generieren.
5. Review und Freigabe durch Stakeholder.

## Best Practices
- Historisierung von kritischen Daten (SCD Type 2).
- Verwendung von UUIDs für verteilte Systeme.
- Klare Trennung von Stamm- und Bewegungsdaten.
