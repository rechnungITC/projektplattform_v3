# Agile Planning

## Übersicht
Dieser Skill unterstützt den Scrum Master und das Entwicklungsteam bei der agilen Planung und Strukturierung von Softwareprojekten.

## Anwendungsbereiche
- Sprint Planning
- Backlog Refinement
- Release-Planung

## Kernkompetenzen
- **User Story Mapping**: Strukturierung von Anforderungen aus Nutzersicht.
- **Story Point Estimation**: Schätzung von Aufwänden (z.B. Planning Poker, Magic Estimation).
- **Akzeptanzkriterien**: Definition klarer "Definition of Done" (DoD).
- **Velocity Tracking**: Messung der Team-Performance über Sprints hinweg.

## Workflow
1. Epics in investierbare User Stories herunterbrechen (INVEST-Prinzip).
2. Akzeptanzkriterien für jede Story definieren.
3. Aufwände im Team schätzen.
4. Stories basierend auf Priorität und Team-Velocity in den Sprint-Backlog ziehen.
5. Sprint-Ziel formulieren.

## Best Practices
- Stories so klein schneiden, dass sie innerhalb eines Sprints abgeschlossen werden können.
- Technische Schulden (Technical Debt) als eigene Backlog-Items erfassen und priorisieren.
